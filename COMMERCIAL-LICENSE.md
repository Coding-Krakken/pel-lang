## What is Dual Licensing?

PEL (Programmable Economic Language) is available under two different licenses:

| License | Best For | Cost | Requirements |
|---------|----------|------|--------------|
| **AGPL-3.0** | Open source projects, research, internal tools | Free | Must share source code when offering as a service |
| **Commercial** | Proprietary products, SaaS offerings, closed-source apps | Paid | No source code disclosure required |

You choose which license fits your use case.

## What Does the AGPL-3.0 Require?

The AGPL-3.0 (Affero General Public License) is like the GPL, but with an important 
network provision:

### Key AGPL Requirements:

1. **Network Use = Distribution**: If you run PEL on a server and let users interact 
   with it over a network (e.g., web service, API, SaaS), you must provide the 
   source code to those users.

2. **Share Modifications**: Any changes you make to PEL must be released under 
   AGPL-3.0 as well.

3. **Compatible Licensing**: If you combine PEL with other software, that software 
   must be AGPL-compatible.

4. **Attribution**: You must retain copyright notices and license information.

### Example Scenarios:

| Scenario | AGPL-3.0 OK? | Need Commercial? |
|----------|--------------|------------------|
| Use PEL internally for your company's financial modeling | ‚úÖ Yes | ‚ùå No (internal use only) |
| Offer a SaaS platform that uses PEL for customers | ‚ùå No* | ‚úÖ Yes (unless you open-source everything) |
| Build a financial app and sell it | ‚ùå No* | ‚úÖ Yes (unless app is AGPL) |
| Use PEL in open source research software | ‚úÖ Yes | ‚ùå No |
| Modify PEL and keep changes private | ‚ùå No | ‚úÖ Yes |
| Bundle PEL with proprietary tools | ‚ùå No | ‚úÖ Yes |
