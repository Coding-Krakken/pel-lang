# PEL Style Guide

This guide defines the default formatting and naming conventions used by `pel format` and `pel lint`.

## Formatting
- Indentation: 4 spaces per block
- Line length: 100 characters (soft limit)
- Spacing: one space around binary operators and after commas
- Braces: K&R style (`model Foo {`)

## Naming
- Models: PascalCase (preferred)
- Parameters: snake_case
- Variables: snake_case

## Comments
- Use `//` for line comments
- Place comments on their own line or at the end of a statement
